# AI 总路线图

这份文档用于统一当前 AI 难度定义、后续演进方向与工程落地顺序。

配套工程拆解见 [ai-plan-search.md](ai-plan-search.md)。

术语约定：

- `牌权续控` 作为文档主术语；在代码、debug 和测试里继续保留 `turnAccess` 作为对应字段名。
- `失先手代价` 指“当前赢轮后很快把牌权送回去”的成本，和 `牌权续控` 配套使用。
- `危险带分领牌` 指“主动首发 `5 / 10 / K` 这类分牌，或 `A / 高主 / 王` 这类高价值高张去试探争轮，但公开信息和 rollout 已显示这手难以稳住牌权，结果既可能送分、又可能白白交掉关键控制资源并立刻送回先手”的高风险动作。
- `高张定门再递牌` 指“朋友已站队后，先用某门 `A` 稳拿一手控制并向同伴传递高张信号，再把同门小牌留给下一拍递牌”的协同动作；它应和危险带分领牌分开判断。
- `朋友已站队` 统一替代 `朋友已明 / 朋友站队` 等混用写法。
- `队友协同` 作为总称；`协同让牌` 视为其中一种具体动作，不再和总称并列使用。
- `埋底 / 保底 / 扣底` 分别指打家重扣底牌、末局守住最后一轮、以及闲家拿到底牌分，不混写。

## 1. 当前原则

- 底牌仅有打家可见，AI 不得把底牌当作公共信息使用。
- AI 不得直接读取其他玩家暗手来做“透视式”决策。
- AI 的甩牌判断必须遵守同样约束：不得用对手暗手直接判断“这手甩牌一定成功 / 一定失败”。
- 在高级 `beliefState` 落地前，甩牌风险只能基于公开信息、已出牌、断门信息和当前难度允许的记牌能力推断。
- 跟牌候选枚举必须优先尝试“直接匹配首家牌型”的结构组合；对中小规模的 `n 选 k` 牌池应完整扫描，不能因为固定组合截断把真实合法跟牌误判成空集。
- 难度差异优先体现在信息利用能力、局势评估深度和搜索能力上，而不是单纯堆 if 规则。
- 当 AI 已经缺首门、又没有选择用主成型去毙时，不能因为另一门刚好成对或成结构，就继续奖励“贴同型”；副牌结构默认应被视为需要保留的后续资源。
- 当 AI 已经缺首门、面对单张跟牌时，也不能为了眼前一手小控制就轻易拆掉 `10+ / 级牌 / 王` 这类大对；若手里还有 `5` 之类可接受的低分贴牌，应默认优先保大对。
- `级牌扣底` 不是单纯的残局特判；尤其在 `J / Q / K / A` 这些特殊级里，它应当被视为能与普通升级并列、甚至更高优先级的整局目标。
- `慢 / 中 / 快 / 瞬` 节奏档位属于交互层配置，只影响 AI 行动等待、发牌与结算过渡；它不参与 AI 难度定义，也不应被拿来掩盖策略问题。
- `重置本局` 图标同样属于交互层配置；它只允许按当前等级重洗当前局并重新发牌，不改变 AI 难度定义、搜索边界或评分逻辑。
- mobile 设置菜单是否重复放置 `重置本局`，也属于同一类交互层编排问题；无论入口保留一处还是两处，都不应进入 AI 路线图范围。
- 结算弹窗里的标题摘要、等级列表和底牌展示属于纯 UI 表达层；这类改动不应反向改写 AI 难度定义、搜索边界和行为策略。
- 经典逐张 SVG、`poker.png`、`m_cards_sprite.png` 这类牌面资源组织方式同样属于表现层；无论默认入口当前指向哪一张整图、`现代整图` 是否开放切换，它们都只能影响渲染与打包，不应进入 AI 评分、记牌或目标判断。
- 即便后续继续微调 `m_cards_sprite.png` 的 tile 对齐方式，让少数窄画布牌面在 mobile 上更贴格，这也仍是表现层工作，不属于 AI 路线图的能力推进项。
- 同理，把整图默认入口在 `poker.png` 与其他实验资源之间切换，也仍然只是表现层配置整理，不属于 AI 路线图里的能力升级项。
- 同理，把原生壳默认入口从 `index2.html` 切到 `index-app.html`，以及把 App 牌桌改成固定顶栏 + 自适应中部出牌区，也只是布局与打包整理，不属于 AI 路线图里的能力升级项。
- 手游手牌标签是否显示张数，也只是移动端信息密度取舍；它不会改变 AI 的公开信息定义，更不属于路线图里的能力演进。
- 同理，手游顶部 `主 / 朋` 状态牌是否统一走 sprite，也只是 UI 渲染一致性问题，不属于 AI 能力推进项。
- `回放种子 / 开局码` 属于调试与复盘基础设施：它们只负责稳定重建开局输入，不能被 AI 当成任何额外信息来源，也不应被算作 AI 能力升级本身。
- headless 回归现在同时保留“同难度全桌”与“`2-3 中级 + 2-3 初级` 混编”两类样本；混编样本可用于暴露中级 AI 的实战缺陷，但不等同于 fixed-seat Elo 评测。

## 2. 当前已实现的难度分层

### 初级

- 核心定位：规则合法 + 固定启发式。
- 记忆能力：不记具体出牌，只使用已经公开暴露出来的断门信息；另外允许记公开已结算的分数归属，用来判断后面剩余分是否还够翻盘。
- 体感目标：像会基本规则、但不会精确数牌的新手。
- 对外描述建议：出牌偏直接，重眼前顺手，不太会专门为后面几轮留牌做局。
- 叫朋友基线：优先找 `副牌 A`，并优先选择自己当前最短的那一门；扣底会尽量围绕这门做短，默认保留 `A + 1 张单牌回手`，优先把这门做成可回手、可毙牌的短门。
- 已补第一版：当自己没有明显主控手，且公开信息已经确认同伴在某门绝门时，初级会把该门小牌当作 `递牌` 门，主动把牌权递给同伴；这条规则同时覆盖“朋友回打家绝门”。
- 已补第二版：朋友已站队后，只要这门还没人公开绝、自己这门顶张是 `A`，并且后面还能留一张小牌时，会优先考虑“先用 `A` 定门、再把后续小牌留作下一拍递牌”；打家侧只有在没有明显主控资源时才会走这条协同线。
- 已补第一版：非打家会做“开局级牌扣底潜力判断”，只基于自己手里的级牌、大主和主长度，决定是否把“级牌扣底”当成轻量副目标；如果后续被叫到朋友但不是叫死，还允许短暂延迟强站队，避免过早把这条路线打断。
- 已补第一版：打家侧增加了“亮友后短窗口优先清主续控 / 无主前几轮少探朋友门 / 朋友晚亮时切 solo fallback”这组首发级 heuristic，用来降低打家在中前盘主动送节奏的概率。
- 已补第二版：`intermediate` 现已把这条路线接进 objective 和统一评分链，不再只靠入口 heuristic；特殊级 `J / Q / K / A` 会进一步提升这条路线的权重。
- 后续补强点：把这条路线从“首发吊主 + 跟牌保资源”继续推进到更完整的末局控轮与扣底兑现。
- 后续补强点：把打家控局补强从首发入口继续推进到跟牌评分和更完整的多轮控局链路。

### 中级

- 核心定位：规则合法 + 更强启发式 + 局部结构意识。
- 记忆能力：记断门；另外会记住和自己当前结构牌有关的高牌是否已经打出。
- 甩牌能力边界：只允许根据已出高张、已暴露断门、当前桌面牌型和自己的结构做“保守甩牌”判断，不得读取未知暗手。
- 体感目标：像会保结构、会看局部高牌的熟练玩家。
- 对外描述建议：出牌开始有布局感，会留对子、连对和关键主牌，明显比初级更会挑时机。

### 高级

- 核心定位：在中级启发式上，开放完整已出牌记忆。
- 记忆能力：记住前面已经打出的全部牌。
- 甩牌能力边界：仍不得读取暗手；只能在“完整已出牌记忆 + 公开行为推断”基础上，做更激进但仍非透视的甩牌判断。
- 体感目标：像会完整记牌、但还没有完整隐藏信息推断的强玩家。
- 对外描述建议：整体节奏更稳，残局更谨慎，更少无谓送掉大牌和关键主牌。

说明：

- 当前产品实际开放的是 `初级 / 中级 / 高级` 三档。
- `高级` 目前是“完整记牌版的中级”，还不是完整意义上的隐藏信息推断 AI。

## 3. 目标中的 4 档定义

为了后续持续升级，推荐把长期目标定义成 4 档：

### 初级

- 规则合法 + 固定启发式 + 少量身份协同。
- 不追求长线规划。
- 重点是稳定、自然、不离谱。
- 可以保留非常少量、非常直白的套路型协同，例如“短门副牌 A 找朋友 -> 用短门回手”。

### 中级

- 单局面评估 + 1 到 2 轮前瞻 + 战略目标权重。
- 开始看“这一手之后会怎样”。
- 目标体感：开始会布局。

### 高级

- 隐藏信息推断 + 多世界模拟 + 风险控制。
- 不再把未知信息当空白，而是维护若干合理猜测。
- 目标体感：开始会读牌。

### 大师

- 搜索树 + 可训练评估函数 + 自博弈迭代。
- 在复杂局面中做近似最优决策。
- 目标体感：开始会算线。

## 4. 中级目标定义

中级的升级重点，不是简单加更多特殊规则，而是从“当前这手最顺”升级到“未来两手更顺”。

### 目标能力

- 首家出牌时，评估“这手打完后剩余结构是否更顺、控牌是否还在、朋友是否更容易被找出”。
- 跟牌时，不只看“能不能压”，还看“这手压了之后牌权会流向哪里”。
- 把 `保底 / 跑分 / 找朋友 / 清某门` 统一成局面目标权重，而不是分散在多个即时判断里。
- 朋友未站队时，持续维护“谁更像朋友”的简单概率分，而不是只靠首轮硬编码。
- 朋友已站队后，中级 AI 应把目标重心从“找朋友推进”切换为“队友协同 / 清主 / 牌权续控 / 保底安全”，避免继续沿用未站队朋友阶段的试探式打法。
- 当自己已经具备明显级牌扣底潜力时，中级应允许 `grade_bottom` 成为正式 objective，并在特殊级里明显提高其权重。
- 中级应把 `递牌` 拆成两段理解：首发时识别“该把牌权递给谁、递哪一门”，跟牌时识别“同伴递牌后我该不该用更大的牌稳接”。
- 朋友未站队时，中级还应识别“前位闲家递高副牌、打家仍在后位”的中位接手窗口，避免把这种牌误判成“暂定闲家互相别抢”而放过抢回主动的机会。
- 当某个候选动作虽然能赢下当前轮，但高概率在下一拍立刻送回牌权时，应对该动作施加强惩罚；`牌权续控` 价值在这类局面中高于单轮小分收益。
- 对甩牌候选，不能调用“全知视角”的成败判断；只能根据已出牌、断门和当前档位允许的记牌范围，评估“这手看起来是否足够安全”。
- 在后程保扣底阶段，要显式理解“是否该提前甩掉王 / 高主，给同侧保留更有效的扣底窗口”，而不是只靠零散启发式。

### 本轮规则复核后新增的中级补齐项

- 增加 `Friend Belief Lite`：未站队阶段持续维护“谁更像朋友”的轻量概率分。
- 把 `甩牌风险` 从候选标签推进成 `evaluateState` 的正式 breakdown。
- 把 `保扣底时的王张释放 / 高主释放` 做成正式评分项，而不是只留在垫牌 heuristic。
- 把 `grade_bottom` 继续从“统一评分已可识别”推进到“更完整的末局兑现与多轮控牌搜索”。
- 继续把 `support-before-reveal / forced reveal / early friend tempo` 这类入口短路规则下沉成评分修正器。
- 把 `递牌` 从“公开绝门回牌”扩展成正式的首发/跟牌协同项，允许中级在公开高张耗尽、后位敌人可能再毙时做更稳的接牌判断。
- 已补一个真实回归口径：
  当前实现会在未站队阶段识别“前位闲家递 `10 / J / Q / K` 一类并不稳控的副牌，且打家仍在后位”的窗口，让中位持有同门 `A / K` 时优先接手，而不是继续按 `tentative_defender_hold` 保守放过。

当前进度补充：

- `Friend Belief Lite` 第一版已开始落地，但还只是“轻量倾向值”，尚未扩展成完整座位概率图。
- `grade_bottom` 第一版 objective 已落地：中级现在会在特殊级和明确潜力局里，把级牌扣底当成正式目标，而不是只沿用普通 `protect_bottom`。
- 共享跟牌短路现已补上一层保护：`保大对先贴牌` 只会在“所有可毙单张都会拆掉受保护高对”时触发；若仍存在不拆高对的安全毙牌，beginner / intermediate 都会继续把这些候选交回正常评分链。

### 基于 2026-03-15 混编回归的近期开发线路

这轮 `20` 局随机 `2-3 个中级 + 2-3 个初级` 的 headless 混编样本已经补跑完毕，结果见 `artifacts/headless-regression/mid-ai-audit-20260315-v3/`。当前结论不是“中级不够强”，而是“中级已经明显强于初级，但在若干关键局面里仍会把本来能稳住的局面重新送回高风险状态”。

本轮样本里最值得优先处理的信号：

- `turn_access_risk` 已选动作仍有 `65` 次。
- `point_run_risk` 已选动作仍有 `44` 次。
- 危险带分领牌仍有 `25` 次。
- `朋友失败` 仍出现 `2` 局，说明未站队阶段的试探式打法依旧偏激进。
- `clear_trump / keep_control` 目标在朋友已站队后命中很多，但部分局面已经出现“控牌目标粘住不放，反而继续领高分高张”的副作用。

据此，近期开发顺序固定为下面 4 步：

1. 先收紧“危险带分领牌”的否决阈值。
   - 重点不是继续给 `dangerousPointLeadPenalty` 加小修小补，而是把它提升成更硬的否决条件。
   - 尤其在 `clear_trump / keep_control` 下，如果候选动作会主动领出 `5 / 10 / K` 这类分牌，或 `A / 高主 / 王` 这类高价值高张，且 rollout 没给出明确的下一拍收益，应优先切回低分安全出法。
   - 验收口径：混编 `20` 局里危险带分领牌次数要显著低于当前 `25` 次。

2. 再修“朋友未站队阶段连续试探高张”的问题。
   - 未站队时，中级不应为了找朋友或维持表面控牌，连续领 `5 / 10 / K` 这类分牌，或 `A / 高主 / 王` 这类高价值高张去试探。
   - 需要增加“高张试探预算”与“下一拍回手保障”判断；没有明确回手保障时，应优先保留高张，转而出更低风险的探路牌。
   - 验收口径：`friend failed` 要从当前 `2/20` 压到 `0-1/20`，且未站队阶段的危险高张首发明显减少。

3. 然后给“朋友已站队后的 clear_trump / keep_control”降温。
   - 当前中级在朋友已站队后，已经会稳定切到 `clear_trump / keep_control / pressure_void`，这本身是对的。
   - 问题在于一旦切到该目标，局部会过度坚持控牌，导致明知有带分风险也继续领高张。
   - 这里需要补一个更明确的“安全退出 / 保底安全 / 低分续控”分支，让 `clear_trump` 只在真正值得清主时保持高优先级。
   - 同时不能把所有 `A` 首发都粗暴视作过热；如果这手 `A` 明确是在“定门给信号，再把下一拍递给同伴”，而公开信息又支持这条续控线，就应保留为正向候选。
   - 验收口径：在朋友已站队阶段，`turn_access_risk + point_run_risk` 的已选次数继续下降，且不再频繁出现连续多拍的高分高张领牌。

4. 最后把混编样本门槛固化为长期回归守门。
   - 当前混编 runner 已经能稳定输出座位难度、逐手事件、分析摘要和异常 seed。
   - 下一步不是只“手工看报告”，而是把关键指标接成可回归的硬门槛。
   - 最少要守住：`20/20` 完局、`dangerous_point_lead` 不回升、`friend failed` 不回升、`turn_access_risk / point_run_risk` 不回升。

这 4 步的优先级不能颠倒：

- 先解决“明显不该出的高风险领牌”。
- 再处理“未站队阶段过早试探”。
- 再做“已站队后的目标切换降温”。
- 最后才把混编指标固化成门禁。

原因：

- 前两步是在修最直接的行为错误，收益最快。
- 第三步是在已有 `control shift` 的基础上做精修，不应在前两步之前展开。
- 第四步属于工程固化，应该建立在前面三步已有稳定下降趋势之后。

### 当前工作区最新校验（2026-03-17）

为了确认路线图判断没有和现时代码脱节，当前工作区又补跑了一轮快速单测和最新 headless 验证，结果已经写入：

- `artifacts/headless-regression/latest/analysis.md`
- `artifacts/headless-regression/latest/mixed-validation/analysis.md`

这轮最新校验给出的补充判断：

- 快速单测 `36 / 36` 通过，说明当前 AI 搜索链路、共享层和 UI 相关回归没有明显回退。
- 同难度全桌 smoke 回归 `3 / 3` 完局、`0` 告警，说明当前状态机和托管流程仍能稳定闭环。
- mobile 顶部托管现已对齐 shared 三态生命周期：`关闭 / 本局托管 / 跨局托管` 只改变玩家是否由 AI 接管与是否跨局保留，不改变 AI 决策强度与规则边界。
- 当前全桌 smoke 样本里，`point_run_risk` 已选动作从上一轮观察到的 `13` 次降到 `9` 次，`dangerous_point_lead` 也从 `4` 次降到 `2` 次，说明“先压危险带分领牌”的方向没有走偏。
- mixed 验证 `2 / 2` 完局、`0` 告警，但当前小样本仍有 `turn_access_risk = 2`、`point_run_risk = 3`、`dangerous_point_lead = 1`，而且样本仍集中在 `friend revealed` 后的 `keep_control / pressure_void / clear_trump` 目标下，说明“朋友已站队后的控牌降温”和“失先手代价正式评分化”依然是未完成主线。
- 当前工作区还额外补了一条“高张定门再递牌”的收口：朋友已站队后，如果打家已经没有明显主控资源，中级和初级都会允许先用副牌 `A` 定门，再把同门小牌留给后续递牌；但如果同伴已经公开绝这门，就会直接回退成低张递牌，不再继续打信号。

这里要特别注意：

- 这轮 mixed 只是一组 `2` 局的当前工作区 smoke，不足以替代上面的 `20` 局混编门槛；其中 `dangerous_point_lead` 仍有 `1` 次，也说明这条风险线还没有真正收平。
- 因此，这轮最新校验只用于确认“优先级没有变”，而不是推翻 `2026-03-15` 那组长期开发顺序。

### 基于 2026-03-16 代码性能审查的新增优先项

这轮补充审查的重点不是“AI 会不会下错”，而是“当前中级搜索链路里，哪些工程开销已经开始反过来限制后续能力继续加深”。

当前新增结论：

- 中级 AI 的主瓶颈已经不再只是权重和规则本身，而是 `每候选 rollout + 多次整局克隆 + 重复评估` 的累计成本。
- 当前性能压力主要集中在 `cloneSimulationState(...)`、`getIntermediateRolloutSummary(...)`、`evaluateState(...)`、`getLegalSelectionsForState(...)` 和 `classifyPlay(...)` 这一条热路径。
- 如果这一层不先收口，后面继续加 `Friend Belief`、`grade_bottom`、更完整残局搜索或更深 rollout，只会把现有卡顿风险继续放大。
- 2026-03-17 已补第一条运行时止血线：
  对 `follow` 模式里的复杂多张跟牌，不能再默认把 shortlist 里的所有候选都送去 rollout；当前实现已改为“先 heuristic shortlist，再按预算决定是否 rollout”，最重的 `5` 张复杂跟牌样本允许直接跳过 rollout。
- 因此，`AI 性能硬化` 现在应被视为中级路线图的正式主线之一，而不是单纯的工程清理。

本轮审查识别出的 4 类主要热点：

1. `每候选 rollout` 都会重复做整局状态深拷贝。
   - 当前中级在首发和跟牌评分里，会为每个候选分别做合法复验、当前轮 rollout，以及必要时的下一拍扩展 rollout。
   - 这会把 `players.hand / played / currentTrick / playHistory / bottomCards` 等字段反复整包复制，候选一多时成本增长很快。

2. 同一个 `simState` 上的派生评估结果缺少复用。
   - `Friend Belief Lite`、`grade_bottom profile`、`turnAccess reserve`、`safeLead summary` 等中间结果，当前会在一次 `evaluateState(...)` 内被不同 breakdown 间接重复构造。
   - rollout 深度一旦扩到 `2`，这些重复成本会被再次放大。

3. 跟牌候选生成仍然偏暴力枚举。
   - `getLegalSelectionsForState(...)` 虽然已经补上“先精确牌型、再组合枚举”的保护，但在部分局面里依然会先枚举大量组合，再逐一校验合法性。
   - 这条路径不仅用于正式决策，也会被 hint、fallback 和 simulation 共用，因此是热点中的热点。

4. 牌型识别与结构检测在热路径里重复过多。
   - `classifyPlay(...)`、`findSerialTuples(...)`、甩牌拆解和一些排序 comparator 里会反复重跑同一手牌的结构识别。
   - 这类开销单次不算最大，但在候选排序、rollout 后评估和 debug 汇总中叠加后很可观。

据此，近期开发顺序在原有“危险带分领牌 / 朋友阶段试探 / 已站队控牌降温”之外，再并行插入下面 4 条性能主线：

1. 先给 rollout 链路做 `state 级缓存与复用`。
   - 目标不是立刻重写算法，而是先把 `baselineEvaluation`、`currentWinningPlay`、`classifyPlay(combo)`、`Friend Belief Lite`、`grade_bottom profile` 这类重复派生信息缓存起来。
   - 验收口径：同一轮同一玩家的候选评分过程中，不再出现明显的重复整包计算。

2. 再压缩 `cloneSimulationState(...)` 的使用次数。
   - 优先减少“同一候选先 clone 校验、再 clone rollout、扩展时再 clone 下一层”的重复调用。
   - 允许保守地保留现有行为语义，但要把“每候选克隆次数”显著降下来。

3. 然后收紧 `getLegalSelectionsForState(...)` 的组合枚举成本。
   - 对 `single / pair / triple / tractor / train / bulldozer` 这类常见首家牌型，优先走结构直达和必要最小候选集，而不是过多回落到通用 `n 选 k`。
   - 验收口径：常见跟牌局面里，候选生成数量和耗时都能稳定控制。

4. 最后把 `牌型识别 memo` 做成共享基础设施。
   - 同一手 `combo` 在同一轮决策中，`classifyPlay(...)`、`getComboKey(...)`、`getFollowStructureScore(...)` 不应在排序器、评估器和 debug 汇总里各算一遍。
   - 这一步做完后，后续继续增加 breakdown 和 rollout 深度才有空间。

这 4 条的优先级说明：

- 它们不是为了替代策略修正，而是为了给后续策略升级腾预算。
- 如果不先做前两条，继续给中级叠更多 rollout / belief / endgame 逻辑，收益会被卡顿和重复计算吞掉。
- 第三条和第四条则是把热点从“整局开销”继续收缩到“候选局部开销”，属于性能收口的第二阶段。

### 推荐实现

1. 把现有 AI 重构成 `候选生成 + 局面打分`。
2. 每个候选动作至少模拟到本轮结束，必要时再模拟到己方下一次出牌机会。
3. 引入统一的 `evaluateState(state, playerId, objective)`。
4. 现有零散 if 规则改成评估项或加权项，而不是直接 `return`。

### 评估项建议

- 手牌结构分：大结构是否保留，碎牌是否减少。
- 控牌分：主牌数、高主数、可回手能力。
- 阵营收益分：己方当前能拿多少分，对方可能跑多少分。
- 找朋友推进分：是否更接近站队，是否避免误打成 `1 打 4`。
- 朋友概率分：当前公开行为下，谁更像朋友，是否仍值得为该方向继续铺路。
- 扣底潜力分：末局时谁更有底牌控制权。
- 扣底协同分：当前是否在为同侧创造更好的扣底窗口，是否在不必要地占住王张和高主。
- 缺门压制分：是否在打对手缺门。
- 甩牌安全分：基于已出高张、未见关键张、断门信息和记牌范围，估计每个组件被压住的风险。

### 工程建议

- `generateCandidatePlays(state, playerId)`
- `simulatePlay(state, play)`
- `simulateTrickToEnd(state)`
- `evaluateState(state, playerId, objective)`

当前的 `chooseAiLeadPlay` / `chooseAiFollowPlay` 逐步退化成：

- 特殊规则修正器
- 候选动作裁剪器

### 成本与风险

- 成本：中等
- 收益：高
- 风险：
  - 状态复制必须轻量化，否则每回合容易卡顿。
  - 评估项过多会互相打架，前期要先少量核心特征。

## 5. 高级目标定义

高级的重点，是从“会看未来”升级到“会猜看不见的牌”。

### 目标能力

- 维护每名玩家的手牌分布假设，而不是把未知信息视为空白。
- 根据亮主、反主、跟牌、垫牌、延迟站队等行为，推断身份和门数。
- 通过多个可能世界评估每个候选动作，而不是只假设一种隐藏信息。
- 让协同开始体现“队友视角”。
- 甩牌判断在这个阶段才允许从“保守记牌判断”升级为“基于 belief 的风险评估”，但仍不能读取真实暗手。

### 推荐实现

1. 增加 `beliefState`。
2. 每次出牌后更新 belief。
3. 基于 belief 采样若干个可能真实牌局。
4. 对每个候选动作，在多个样本上做短搜索并取均值或风险加权值。

### beliefState 可先记录

- 每名玩家各门剩余张数范围
- 持有朋友牌概率
- 持有高主 / 对子 / 拖拉机概率
- 更像打家方还是防守方

### 推荐接口

- `buildBeliefState(state, observerId)`
- `sampleWorldsFromBelief(state, observerId, count)`
- `evaluatePlayAcrossWorlds(state, play, worlds)`

### 成本与风险

- 成本：高
- 收益：非常高
- 风险：
  - 实现复杂度明显上升
  - 参数多，调参成本高
  - 样本不足时容易“猜错但很自信”

## 6. 大师目标定义

大师阶段不再是“更复杂的 if”，而是进入“搜索 + 评估器 + 训练”的体系。

### 目标能力

- 基于搜索树寻找局部最优线。
- 评估函数从纯手写特征，逐步升级为可训练模型。
- 根据局面自动切换风格：
  - 跑分局
  - 找朋友局
  - 保底局
  - 拖控局
  - 骗主局
- 能通过自博弈持续迭代。

### 推荐架构

- 规则引擎：合法动作、胜负判定
- 候选生成器：裁掉明显低价值动作
- 隐藏信息采样器：生成 plausible worlds
- 搜索器：展开若干层
- 评估器：为叶子局面打分
- 策略管理器：根据阶段调节风险偏好

### 搜索方向

- Monte Carlo Tree Search
- Beam Search
- 有限深度 Alpha-Beta 风格搜索

### 评估特征建议

- 我方控牌指数
- 敌方可能主力指数
- 剩余分牌暴露度
- 扣底威胁度
- 朋友显性 / 隐性协同度
- 各门可推进性
- 当前牌权转移链长度
- 关键高牌保留价值

### 成本与风险

- 成本：极高
- 风险：
  - 工程量最大
  - 需要大量仿真与对局日志
  - 候选裁剪做不好时，搜索分支会爆炸

## 7. 推荐实现顺序

最稳妥的顺序是：

1. 统一框架
   把现有规则重构成“候选生成 + 评分项”。
2. 做中级前瞻
   先跑通 1 到 2 轮短视搜索，这是性价比最高的一步。
3. 做高级 belief
   再引入隐藏信息推断和多世界模拟。
4. 最后冲大师
   搜索和训练必须建立在前两层稳定之后。

## 8. 验收标准

- 中级：开始会布局
- 高级：开始会读牌
- 大师：开始会算线

## 9. 建议专项维护模块

当前已经单独维护的专项：

- `出牌搜索 / 局面评估`
- `亮主 / 反主`

下一批最值得单独维护的专项：

- `拿底 / 埋底 / 保底`
  这部分和出牌搜索强相关，但决策窗口、约束条件和验收标准都很独立，适合像搜索和亮主一样单独维护。
- `找朋友 / 站队协同`
  这部分直接决定“朋友未站队”和“朋友已站队后”的策略切换，也值得单独维护目标、回归样本和调参口径。

建议暂时不要再拆成独立文档，但要持续作为横切能力维护的模块：

- `记牌 / 信息利用`
  这是中级和高级的共同基础，更适合作为搜索、甩牌风险、亮主读牌等模块的共享能力层。

## 10. 当前开发约束

- 现阶段优先保证：
  - 出牌自然
  - 不透视
  - 可解释
  - 回归稳定
- 任何甩牌保护、甩牌过滤、甩牌评分都必须满足“不看对手暗手”这条红线。
- 如果当前实现里某个 helper 通过真实 `state.players[x].hand` 判断甩牌成败，它只能属于规则裁定层，不能直接用于 AI 决策层。
- 在 `中级` 搜索框架落稳之前，不建议过早引入复杂 hidden sampling。
