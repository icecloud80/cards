# 五人找朋友 App 首发版本任务拆解清单

这份文档把 `v1.0 单机 App 首发` 需要完成的工作拆成可排期、可分工、可验收的任务清单。

配套文档：

- 产品需求见 [mobile-app-product-requirements.md](mobile-app-product-requirements.md)。
- 技术设计见 [mobile-online-architecture.md](mobile-online-architecture.md)。
- 阶段路线图见 [mobile-online-roadmap.md](mobile-online-roadmap.md)。

## 1. 本文档目标

- 把“做成 iOS / Android App”从方向性规划落到执行层。
- 明确哪些任务属于 `v1.0` 必做，哪些可以延后。
- 明确每类任务的依赖关系、验收口径与主要风险。
- 让产品、设计、开发、QA 可以基于同一份清单推进。

## 2. 首发版本目标重述

`v1.0` 的目标不是做多人在线，而是做出一个稳定、可发布、可监控、可变现的单机 AI 对战 App。

首发必须满足：

- 支持 `iOS / Android` 真机运行。
- 支持完整单机对局流程。
- 支持本地进度保存与继续游戏。
- 支持基础广告、埋点、崩溃监控、公告和远程配置。
- 支持 App 商店提审与正式发布。

## 3. 任务优先级定义

- `P0`：首发阻断项，不完成不能上线。
- `P1`：强烈建议首发完成，影响体验与数据质量。
- `P2`：可在首发后快速补齐，不阻断上线。

## 4. 工作流总览

首发任务按 8 个工作流推进：

1. 产品与范围冻结。
2. 客户端 App 壳与构建链。
3. 存档与本地数据迁移。
4. UI / UX App 化改造。
5. 原生能力接入。
6. 广告与运营基础设施。
7. QA 与发布准备。
8. 上线后监控与回滚预案。

## 5. 任务拆解

### 5.1 产品与范围冻结

#### 任务 1：冻结 `v1.0` 功能范围

- 优先级：`P0`
- 输出：
  - 首发必须功能列表。
  - 首发明确不做列表。
  - 发布验收口径。
- 依赖：无。
- 验收：
  - 产品、开发、设计对 `v1.0` 范围达成一致。
  - 不再把联机、排位、社交强行塞入首发。

#### 任务 2：定义首发 KPI

- 优先级：`P1`
- 输出：
  - 首局完成率目标。
  - `D1 / D7` 留存观察口径。
  - 广告展示率和激励完成率观察口径。
- 依赖：任务 1。
- 验收：
  - 上线后能明确判断版本是“可继续放量”还是“先修体验”。

### 5.2 客户端 App 壳与构建链

#### 任务 3：建立 App 工程

- 优先级：`P0`
- 输出：
  - iOS 壳工程。
  - Android 壳工程。
  - 本地调试与打包说明。
- 依赖：任务 1。
- 验收：
  - 能在真机安装运行当前项目。
  - 能区分开发 / 测试 / 正式环境。
- 当前初始化约定补充：
  - App Name：`找朋友升级`
  - App ID：`com.nolanli.cards`
  - `Capacitor webDir`：`dist/app`
  - 默认 App 入口：`dist/app/index.html -> index-app.html`
  - Android 可先独立推进；iOS 完整同步除 `CocoaPods` 外，还要求本机有完整 Xcode 环境

#### 任务 4：建立正式构建流程

- 优先级：`P0`
- 输出：
  - 静态资源打包流程。
  - 版本号管理方案。
  - 环境变量方案。
  - App Icon 统一生成命令与资源验收口径。
- 依赖：任务 3。
- 验收：
  - 能稳定生成提审包。
  - 资源版本可追踪。
  - App Web 默认资源清单不再包含 `elements.cardmeister.min.js` 这类废弃脚本，并有单测校验最终 `dist/app` 产物不回退。
  - App Icon 需支持通过 `npm run app:icons` 一次性产出 iOS 1024 图标、Android legacy icon、round icon 与 adaptive foreground，避免双端资源手工漂移。

#### 任务 5：建立 App 端 smoke 检查

- 优先级：`P1`
- 输出：
  - 冷启动检查项。
  - 开局到结算的真机 smoke 清单。
- 依赖：任务 3、任务 4。
- 验收：
  - 每次提测前都能快速验证主流程未坏。

### 5.3 存档与本地数据迁移

#### 任务 6：梳理当前浏览器持久化数据

- 优先级：`P0`
- 输出：
  - 当前 `cookie / localStorage` 字段清单。
  - 新存储结构映射表。
- 依赖：任务 1。
- 验收：
  - 明确哪些数据必须迁移，哪些可以放弃。

#### 任务 7：接入 App 端持久化

- 优先级：`P0`
- 输出：
  - 设置存储。
  - 进度存储。
  - 最近一局复盘输入存储（`回放种子 + 开局码`）。
- 依赖：任务 6。
- 验收：
  - 关闭 App 重开后可恢复关键设置和进度。
  - 不再依赖浏览器 cookie 才能继续游戏。
  - 最近一局可继续用于日志、QA 和手动重建开局。

#### 任务 8：实现存档迁移与兜底

- 优先级：`P1`
- 输出：
  - 旧存档迁移逻辑。
  - 读档失败回退逻辑。
- 依赖：任务 7。
- 验收：
  - 异常存档不会导致 App 卡死或进度全部不可用。
  - 这轮不要求恢复到中断时刻继续打，只要求最近一局复盘输入不丢。

### 5.4 UI / UX App 化改造

#### 任务 9：重做首页为 App 入口

- 优先级：`P0`
- 输出：
  - App 首页结构稿。
  - 开始游戏 / 继续游戏 / 规则 / 设置入口。
- 依赖：任务 1。
- 验收：
  - 首页不再像网页原型入口拼装。
  - 新玩家一眼能知道怎么开始。

#### 任务 10：整理局内顶部状态与设置入口

- 优先级：`P1`
- 输出：
  - 顶部状态块精修稿。
  - 局内设置页或设置弹层。
- 依赖：任务 9。
- 验收：
  - 主牌、阶段、轮到谁、闲家总分都能快速读到。
  - 设置入口不遮挡主战场。

#### 任务 11：优化手牌操作体验

- 优先级：`P0`
- 输出：
  - 选牌反馈。
  - 误触撤销。
  - 长按预览或更清晰的选中态。
- 依赖：任务 3。
- 验收：
  - 手游长局不会频繁因为误触影响体验。

#### 任务 12：整理关键阶段面板

- 优先级：`P1`
- 输出：
  - 亮主 / 反主。
  - 扣底。
  - 叫朋友。
  - 结果页。
- 依赖：任务 9、任务 10。
- 验收：
  - 关键阶段的按钮和信息更聚焦。
  - 新手更容易理解当前要做什么。

### 5.5 原生能力接入

#### 任务 13：接入崩溃监控与错误上报

- 优先级：`P0`
- 输出：
  - 崩溃监控 SDK。
  - JS 错误上报。
  - 版本与设备信息带入。
- 依赖：任务 3、任务 4。
- 验收：
  - 测试环境可看到真实崩溃和错误日志。

#### 任务 14：接入埋点

- 优先级：`P0`
- 输出：
  - 关键事件埋点表。
  - 首局漏斗。
  - 广告埋点。
- 依赖：任务 1、任务 4。
- 验收：
  - 能追踪从启动到首局完成的关键流失点。

#### 任务 15：接入远程配置与公告

- 优先级：`P1`
- 输出：
  - 远程配置字段表。
  - 公告投放位。
- 依赖：任务 14。
- 验收：
  - 不发版本也能调整广告频率和开关。
  - 可在线发布公告。

#### 任务 16：处理 App 生命周期

- 优先级：`P0`
- 输出：
  - 切后台冻结。
  - 回前台恢复。
  - 崩溃后恢复未完成对局。
- 依赖：任务 7。
- 验收：
  - 来电、锁屏、切后台不导致明显丢局。

### 5.6 广告与运营基础设施

#### 任务 17：确定首发广告位方案

- 优先级：`P0`
- 输出：
  - 插屏触发时机。
  - 激励视频入口。
  - 免广告冷启动策略。
- 依赖：任务 1、任务 2。
- 验收：
  - 广告位不进入牌桌主流程。

#### 任务 18：接入广告 SDK

- 优先级：`P0`
- 输出：
  - 插屏广告。
  - 激励视频。
  - 加载失败回退。
- 依赖：任务 17、任务 3。
- 验收：
  - 广告展示、关闭、失败都不会卡住游戏流程。

#### 任务 19：实现广告频控

- 优先级：`P1`
- 输出：
  - 局间插屏冷却。
  - 每日上限。
  - 新手保护期。
- 依赖：任务 15、任务 18。
- 验收：
  - 首局完成率不因广告过度打扰而明显受损。

#### 任务 20：设计首发激励奖励

- 优先级：`P1`
- 输出：
  - 非公平性奖励清单。
  - 激励视频入口文案。
- 依赖：任务 17。
- 验收：
  - 奖励不会影响对局公平性。

### 5.7 QA 与发布准备

#### 任务 21：整理真机兼容矩阵

- 优先级：`P0`
- 输出：
  - iPhone 机型清单。
  - 安卓中高低端机清单。
- 依赖：任务 3。
- 验收：
  - 首发测试覆盖主流机型与关键屏幕尺寸。

#### 任务 22：建立首发回归清单

- 优先级：`P0`
- 输出：
  - 新开局。
  - 继续游戏。
  - 托管。
  - 结果页。
  - 广告回调。
  - 切后台恢复。
- 依赖：任务 5、任务 16、任务 18。
- 验收：
  - 每个提测包都可执行同一套回归。

#### 任务 23：准备商店素材

- 优先级：`P0`
- 输出：
  - 图标。
  - 截图。
  - 描述文案。
  - 隐私政策与年龄分级材料。
- 依赖：任务 9、任务 12。
- 验收：
  - 可直接进入商店提审流程。
  - 图标需和 App 首页主视觉共用同一套品牌口径，并保留可重复生成脚本与预览图，避免提审前临时手工导出一版后无法复用。

#### 任务 24：准备发布与回滚预案

- 优先级：`P1`
- 输出：
  - 分阶段放量方案。
  - 广告与公告紧急开关。
  - 回滚与热修优先级清单。
- 依赖：任务 13、任务 14、任务 15、任务 18。
- 验收：
  - 上线出问题时能快速止损。

### 5.8 上线后监控与复盘

#### 任务 25：建立上线后 7 日监控面板

- 优先级：`P1`
- 输出：
  - 崩溃率。
  - 首局完成率。
  - 广告展示率。
  - 激励完成率。
  - 留存趋势。
- 依赖：任务 14、任务 24。
- 验收：
  - 上线后可在每天固定节奏复盘。

#### 任务 26：建立首发 bug 清单与分级机制

- 优先级：`P1`
- 输出：
  - `P0 / P1 / P2` bug 定义。
  - 日报与修复节奏。
- 依赖：任务 22、任务 25。
- 验收：
  - 团队能快速判断先修什么。

## 6. 任务依赖关系

### 6.1 最短关键路径

下面这条链条不建议打乱：

1. 冻结首发范围。
2. 建立 App 工程与构建流程。
3. 完成本地存储迁移。
4. 完成首页和关键阶段 UI App 化。
5. 接入监控、埋点、广告和远程配置。
6. 完成真机回归、商店素材与发布预案。

### 6.2 可并行推进

- UI 设计稿与商店素材可并行准备。
- 崩溃监控、埋点、远程配置可以和广告接入并行。
- QA 兼容矩阵可以在 App 壳可运行后尽早开始。

## 7. 首发验收清单

首发提审前至少满足：

- iOS / Android 两端都能完整打完一局。
- 设置、进度、继续游戏正常。
- 切后台恢复正常。
- 插屏与激励视频成功与失败都不阻断流程。
- 崩溃监控、埋点、远程配置在线可用。
- 商店素材、隐私政策、年龄分级材料齐备。

## 8. 建议排期

如果资源有限，建议按下面顺序推进：

### 第一周

- 范围冻结。
- App 工程初始化。
- `Capacitor` 配置落盘，固定 `找朋友升级 / com.nolanli.cards / dist/app`。
- 存储迁移方案。
- 首页与关键阶段 UI 稿。

### 第二周

- App 壳打通。
- 本地存储接入。
- 关键 UI 页面改造。
- 崩溃监控与埋点接入。

### 第三周

- 广告接入。
- 远程配置与公告。
- 切后台恢复。
- 真机兼容问题第一轮修复。

### 第四周

- 全量回归。
- 商店素材。
- 发布预案。
- 灰度上线准备。

## 9. 首发后立即接的下一批任务

`v1.0` 上线稳定后，建议立刻接下面 4 组任务：

1. 登录与云存档。
2. 新手引导优化。
3. 激励奖励与轻外观体系。
4. 好友房前置技术准备。
