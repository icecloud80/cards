# 手游静态牌桌模板说明

这份文档记录 `index2-static.html` 的设计目的、交互范围和维护约束，作为 `index2.html` 运行态之外的独立 mobile 视觉验收基准。

## 1. 目标

- 提供一个不依赖共享运行时的手游静态牌桌页。
- 让评审、截图、样式调试时可以直接打开固定局面，而不必先从开始页进入真实对局。
- 保留和 `index2.html` 同口径的关键结构：顶部状态、中央 5 个出牌位、底部手牌、底部按钮区和设定菜单。
- 把 `信息 / 上一轮 / 底牌 / 规则帮助 / 牌面切换 / 复制复盘码 / 返回首页` 做成真实可点的轻交互，方便一次性验证多个 UI 状态。

## 2. 固定样例范围

- 顶部固定展示 `计秒 / 总分 / 难度 / 主牌 / 朋友牌 / 托管`。
- 中央固定展示 5 个出牌位，其中包含打家、朋友和自己托管态样例。
- 底部手牌区固定展示 25 张手牌，覆盖 `主牌 / 方块 / 梅花 / 黑桃` 四组。
- 底牌面板固定展示 7 张底牌。
- 信息面板固定展示最近播报与 5 位玩家摘要。
- 上一轮回看固定展示 5 位玩家的完整上一轮牌列。

## 3. 交互约束

- 这页只允许有展示层交互，不接入真实出牌、AI、计时或存档逻辑。
- 菜单里允许切换牌面主题，默认使用 `poker.png`，并保留 `classic` 单张牌面作为对照。
- 菜单里的 `复制复盘码` 需要固定携带一组 mock `回放种子 + 开局码`，方便评审跨端复盘按钮的位置与反馈文案。
- 当前 mock 口径必须和真实运行态一致：`回放种子` 固定 `11` 位、`开局码` 固定 `169` 位，且开局码区分大小写。
- 顶部托管按钮继续保留 `关闭 / 本局托管 / 跨局托管` 三态高亮，只做视觉切换，不触发真实托管。
- `返回首页 / 开始游戏 / 继续游戏` 只负责 `setup / play` 两种静态画面切换。
- 开始页品牌标题需与运行态和 App 壳页统一固定为 `找朋友升级`，避免静态模板继续保留旧品牌名导致评审口径分叉。

## 4. 与真实玩法的边界

- 不修改规则。
- 不修改叫朋友、扣底、跟牌或计分逻辑。
- 不修改 AI 策略、AI Heuristic 或难度定义。
- 不替代真实运行态回归；它的职责是补一个稳定的 mobile 视觉样本。

## 5. 维护要求

- 若 `index2.html` 的 mobile 结构发生明显变化，需同步评估 `index2-static.html` 是否要跟进。
- 若菜单入口、顶部状态卡或手牌区结构调整，需同步更新静态模板和对应回归测试。
- 当前静态页回归测试为 [check-mobile-static-template.js](../tests/unit/check-mobile-static-template.js)。
- 预览时统一使用本地 HTTP 服务，通过 `/index2-static.html` 打开，不建议继续双击 `file://` 方式验页面。
