# PC 界面 v2 原型需求

这份文档记录 2026-03-15 这一轮 PC 桌面端原型改版目标，作为设计稿和后续实现的共同参照。

## 1. 改版背景

- 旧版 PC 原型把玩家信息、功能按钮和说明文案都摊在桌面上，阅读负担偏重。
- mobile 端的思路更接近“先看局面，再看动作”，信息优先级更清晰。
- 这次 PC 改版不改玩法规则，重点只在桌面信息架构和原型布局。
- 像“缺首门贴副时优先保副牌结构”“打家先出朋友牌而朋友延迟站队时仍优先保同门拖拉机”这类 AI 规则修正，属于共享策略层变更，不要求 PC 额外新增独立交互入口。
- 同理，后续像“叫朋友改成 `A -> K -> 找朋友牌`”“`A` 级改叫 `K`、先走 `Q`”“副牌过脏时改叫第一张大王”这类共享 AI 规则修正，也都属于共享策略层变更，不要求 PC 额外新增独立交互入口。

## 2. 本轮目标

- 参考 mobile 的轻量信息结构，减少 PC 长句说明。
- 把不重要或低频的功能收进菜单或按钮，不再长期占用主战场。
- 重新规划 5 个出牌区，让桌面构图更规整、更对称。
- 把头像、身份、等级、剩余牌数、个人分数等关键信息并入出牌区头部。
- 简化独立玩家面板，不再承担“展示手牌”的任务。

## 3. 新版布局原则

### 顶部

- 顶部只保留一行高优先级状态：
- 本局
- 主牌
- 朋友
- 阶段
- 当前轮到谁
- 倒计时
- 闲家总分

- 顶部状态和右上菜单并入同一个整体容器。
- 倒计时和闲家总分放在顶部最左侧，作为第一视觉信息。
- 计时与总分都用短标签显示，避免顶部出现大段字面说明。
- `计时 / 总分` 标签放到各自胶囊上方，胶囊内部只保留数字本体。
- 朋友牌缩成顶部里的紧凑状态块。
- `朋友` 状态块里的出现进度改成 `1/2` 这类紧凑写法，替代“第几张”长文案。
- PC 的“找朋友”弹层文案统一改成“叫朋友”，避免顶部阶段文案和弹层标题继续混用两种叫法。
- 打家进入“叫朋友”后，需要在面板里看到 30 秒读秒；`用推荐` 按钮直接带剩余秒数，超时会自动确认当前选择。
- 打家首次确认后，在同一轮读秒结束前，还能点击顶部朋友牌再编辑一次；这次机会只能用一次，用掉后恢复普通首手读秒。
- `游戏状态 / 当前首家` 这类提示移入 `局` 状态块，不再单独挂在游戏名下方。
- `主` 状态块增加主牌可视徽章，和 `朋` 的可视锚点保持统一。
- PC 顶部 `主牌` 状态块继续对齐手游：优先展示“实际亮出的那张牌”，无主时也要显示触发亮主的王，而不是只剩一个 `无` 字。
- PC 顶部 `主牌` 状态块第二行的打家提示统一写成 `打家 玩家X`，不再缩成容易误读的 `打 玩家X`。
- PC 顶部 `主牌` 牌面卡位尺寸继续对齐 `朋友` 牌面卡位，且不再保留外层橘色描边或米色卡框。
- PC 顶部 `朋友` 状态块在 `已站队` 后要直接带出玩家名，例如 `已站队 玩家4`，避免只看到状态却不知道是谁已经站队。
- PC 顶部 `朋友` 牌面卡位同样不再保留橘色描边，主牌与朋友都统一成纯牌面展示。
- 顶部去掉游戏标题，只保留状态信息和右上菜单。
- 规则帮助、Debug、播报、底牌、布局编辑、AI 难度、牌面切换等放进 `更多功能` 菜单。
- 顶部不承担新手导引职责，只保留短状态，不放规则解释句。
- 顶部 5 个功能按钮固定为单排图标：`托管 / 信息 / 回看上轮 / 刷新 / 设置`。
- 顶部 5 个按钮统一使用提供的图标素材，其中 `刷新` 固定使用仓库内的 [icon-refresh.png](../images/icons/icon-refresh.png)。
- `刷新` 只负责按当前 `回放种子` 重开这一局：重新发牌、清空本局过程状态，但不把长期等级清回 `2`，也不重新分配新 seed。
- PC 顶部 `信息` 与 `上一轮回看` 的下拉面板，顶部起点需要和 `更多功能` 菜单保持同一条水平线，避免同属 topbar 的二级层出现上下错位。
- `上一轮回看` 抽屉内的 5 位玩家记录需要改成紧凑横排行：
  - 左侧固定显示玩家名与身份短签
  - 右侧使用横向重叠牌列展示上一手
  - 目标是减少抽屉纵向高度，避免上一轮信息把主战场下压过多
- PC 端在 `快 / 瞬` 节奏下也不能因为高级 AI 的复杂多张跟牌而出现明显停顿；这类卡顿应优先通过 AI 搜索预算收口解决，而不是靠继续降低 UI 等待或隐藏播报掩盖。

### 中央战场

- 中央区域作为 PC 第一视觉焦点。
- 5 个玩家出牌区统一为同规格卡片，避免旧版左大右小、上下松紧不一致的问题。
- 每个出牌区头部直接显示：
- 头像
- 玩家名
- 阵营身份
- 等级
- 剩余牌数或个人分数
- 特殊状态，例如 `大`、`打`、`朋`
- 等级位宽要兼容 `Lv:-10`、`Lv:-K` 这类更长负级文本，不能只按旧的 `Lv:-A` 预留空间。

- 未出牌时显示短空态，例如 `待出`，不再写长说明。
- 其他玩家出牌区标题去掉“出牌区”三个字，只保留玩家名本身。
- 出牌区默认不再显示“本轮尚未出牌”这类空态提示，让中央留白更干净。
- 出牌区里的 `打 / 朋` 身份改成和手游一致的底色短签，不再用长副标题解释。
- PC 出牌区标题行里的身份短签进一步改成完整写法：显示 `打家 / 朋友`，不再缩成单字 `打 / 朋`。
- `托管` 胶囊并到出牌区标题行，直接贴在 `打 / 朋` 身份短签旁边，不再单独挂在右侧。
- 出牌区里的主牌卡不再使用黄色描边强调。
- 中央保留一条焦点提示带，用于关键操作提示和重要播报。
- 中央桌面背景整体右移，避免和左侧玩家面板重叠。

### 玩家面板

- 玩家面板退化成轻量状态列。
- 5 个玩家面板统一移动到左侧，形成连续纵向状态列。
- 默认只保留头像、序号、身份或轮转信息。
- 去掉“谁操控”这类说明文字。
- 阵营统一改成 `打 / 闲 / 朋` 三种紧凑图标，未知阵营直接留空。
- 左侧玩家面板整体背景改成更轻的淡色玻璃感，不再使用厚重底色。
- 玩家1 面板做成更大的专属卡片，额外显示自己的短状态，而不是沿用其他玩家的同款模板。
- 玩家1 左侧面板最终收成和其他玩家同宽、但高度做成正方形的专属卡片，和玩家2之间的垂直节奏保持一致。
- 左侧玩家面板顺序固定为从上到下 `玩家2 / 玩家3 / 玩家4 / 玩家5 / 玩家1`。
- 不再在独立玩家面板里展示手牌内容。
- 需要展示的关键状态，优先合并到对应的出牌区头部。

### 底部

- 底部改成“左手牌区、右操作区”的双栏结构。
- 左侧整块交给手牌分组展示，取消手牌区里的操作说明文案。
- 底部手牌区去掉头像、标题和 `Lv / 手牌数` 等头部信息。
- 右侧操作区只保留按钮，不放阶段说明和选牌提示文案。
- 各花色分组增加数量徽标，不显示滚动条，改为通过更大的牌面重叠来容纳长手牌。
- 常态下按约 `20% ~ 25%` 的可视牌面密度压叠；拿起底牌后再动态加大重叠，保证 31 张牌仍尽量留在同一视野内。
- 底部手牌区移除整块浅色背景，让牌组直接贴近桌面展示。
- 左侧手牌区的各花色分组紧贴排列，不再在花色之间额外留空。
- 底部手牌区整体向右收，并补一圈轻线框，避免太贴左边且缺少边界。
- 底部手牌区改为居中加宽，并把“主牌 / 花色统计”挪到左侧竖排统计列里，减少横向占位。
- 底部手牌区继续精修为“按钮在牌上方、统计在牌下方”的三层结构，`主牌 / 花色` 统计改成贴着对应牌段显示的白字短签。
- 高频动作继续贴近手牌区右侧保留，形成独立操作区。
- 右侧操作区恢复偏蓝的配色，和中央桌面冷色环境更统一。
- 右侧操作区去掉外层面板背景，按钮直接落在桌面右侧区域。
- 右侧操作按钮整体下沉，并放大按钮尺寸与蓝色对比度，让操作区更容易被看到。
- 亮主阶段需要把当前所有合法方案直接做成可点击按钮，不再保留“先选方案，再点亮主”的二次确认。
- 操作按钮进一步并入手牌区上方，和选牌区域形成同一块操作范围。
- 手牌区、按钮区和中央桌面背景继续按同一套冷色玻璃层级做细调，重点收口边框、阴影和留白比例。
- 中央出牌区去掉玩家头像，只保留名字、身份短签和右上角金色 `大` 角标；牌面略缩小并改成居中摆放。
- 出牌区标题行里的 `托管` 胶囊已经并到身份短签旁边，取消右侧单独指标位。
- PC 操作区进一步减法，移除 `毙牌 / 开始游戏 / 继续游戏` 这些不再需要的局内按钮。
- 版本号最终收回到右下角；手牌区只保留外层边框，按钮贴进框内上沿并缩小；玩家面板字体与颜色回到更清晰的浅色口径；中央出牌区整体再向右收一档，确保完全落在台面框内。
- 手牌区继续加高，按钮再压扁一档；花色统计缩小并稳定留在手牌框内；中央出牌区整体再上提，进一步落回台面框内部。
- 手牌区底部花色统计继续精修为“和对应牌段首张牌左沿对齐，并紧贴牌底边”。
- PC 连续牌轨在 31 张长手牌场景下，首张牌必须完整保留在手牌框内，不能因为重叠样式把最左边牌角裁掉。
- 玩家面板里的玩家名统一回到普通 sans-serif 字体，避免和整体原型风格脱节。
- 左侧玩家面板里的 `打 / 闲 / 朋` 短签改成更高对比度的深底亮字，避免在桌面背景上发灰。
- PC 与静态模板当前默认继续使用 `poker.png`，同时新增 `m_cards_sprite.png` 对应的 `现代整图` 主题供切换。
- 操作按钮继续对齐到手牌框内部上沿右侧，和静态模板保持一致。
- PC 运行态的操作区不再单独锚在右侧战场边缘，而是重新收回到底部手牌区的中上部。
- 顶部图标按钮的状态文案改成只更新 `aria-label / title`，不能再用 `textContent` 覆盖掉内部图标。
- 底部花色统计继续贴近牌底边，手牌框与中央桌面背景之间保留清晰分隔，不再相互压边。
- 新增独立静态模板页 `index-static.html`，用 mock data 固定展示顶部、左侧玩家列、中央战场和 31 张手牌，专门用于 PC 视觉对齐。
- 静态模板页里的 `信息 / 上一局回看 / 设置菜单 / 切换牌面 / 重置本局 mock 切换` 都可直接点击，便于连同交互状态一起做视觉验证。
- 静态模板页和运行态继续共用同一套底部规则：花色统计按首牌对齐、按钮落在手牌框内上沿、默认牌面走 sprite。
- 大改动提交流程会追加真实浏览器 UI smoke：自动打开 `index1.html` 与 `index2.html`，统一切到 `瞬` 档并开启托管，确认 PC / mobile 都能从真实入口顺利打完一局。
- App 专用原生页现已独立到 `index-app.html`；PC 设计稿与桌面运行态不需要跟着承担 iOS / Android 安全区补丁，继续只维护桌面端信息架构与视觉规则。
- PC 牌面切换需要允许接入单张资源目录之外的整图 sprite；当前运行态默认继续使用仓库根目录的 `poker.png`，并新增 `m_cards_sprite.png` 的 `现代整图` 主题，同时继续保留 `classic` 单张 SVG 作为对照与兜底。
- `index-static.html` 默认直接使用 `poker.png` 作为牌面主题，顶部总分先用 `125` 占位，作为视觉验收基准。
- 翻底定主阶段未翻开的底牌也优先使用当前统一 sprite 里的牌背，避免和正面牌风格脱节。

### 开始界面

- 开始界面保留 `开始游戏 / 继续游戏 / 查看规则` 三个主入口。
- PC 的 `查看规则` 与 mobile / App 的 `规则帮助` 继续共用同一版局内快读文案，避免桌面端和移动端各自维护不同的规则摘要口径。
- `AI难度` 从局内顶部移到开始界面设置。
- 开始界面与局内设置都提供 `对局节奏` 选择，统一支持 `慢 / 中 / 快 / 瞬` 四档。
- 开始界面主视觉直接复用手游开始页插画，统一保留 `WELCOME + 扑克牌扇形 + 拖拉机 / 火车` 这组入口氛围元素，不再单独维护一张 PC 静态图。
- PC 顶部状态条左侧统计补入与手游同口径的 `难度` 短标签，默认展示 `初 / 中 / 高` 这类紧凑写法，减少跨端口径差异。

## 4. 与 mobile 的对齐点

- 都把公共状态压缩成短标签和短句，不依赖大段说明文案。
- 都强调中央战场的当前局势，而不是边栏堆信息。
- 都优先展示行动相关信息，再展示解释性信息。
- 都把低频功能收纳，不抢主流程注意力。
- 都把结算结果压缩成“标题即结果 + 逐人等级列表”的短读法，不再依赖胶囊标签和长段等级说明。
- 逐人等级列表继续沿用你的文字结构，但视觉上升级成“玩家名 + 阵营胶囊 + 等级箭头 + 结果胶囊”的卡片行，减少纯文本堆叠感。
- 逐人等级卡片继续细化为“阵营与结果胶囊靠左、等级胶囊靠右”的信息分区，并用整行绿/红底直接表达升降级。
- 手游发牌阶段的声明区同步走紧凑化思路：缩短手牌区高度、减小牌面尺寸，并把可亮候选改成横向滚动 chips，避免内容超出屏幕。
- 手游发牌亮主阶段继续减法：进入有候选的声明选择态后，不再保留额外操作按钮，声明 chips 直接成为唯一主交互，并用花色图标压缩候选宽度。
- 声明 chips 继续从“图标化文案”收紧为“真实牌面叠图”，例如两张大王、两张方片 2 都直接显示缩略牌堆，默认约 75% 重叠。
- 手游翻底定主公示层单独走轻浮层精修：标题区增加阶段短标签，底牌轨道整体上提；底牌下方放置类似 `关闭 (12s)` 的读秒胶囊按钮，panel 右上角再补独立 `X`，减少关闭入口和读秒提示互相抢位。
