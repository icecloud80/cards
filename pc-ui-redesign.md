# PC 界面 v2 原型需求

这份文档记录 2026-03-15 这一轮 PC 桌面端原型改版目标，作为设计稿和后续实现的共同参照。

## 1. 改版背景

- 旧版 PC 原型把玩家信息、功能按钮和说明文案都摊在桌面上，阅读负担偏重。
- mobile 端的思路更接近“先看局面，再看动作”，信息优先级更清晰。
- 这次 PC 改版不改玩法规则，重点只在桌面信息架构和原型布局。

## 2. 本轮目标

- 参考 mobile 的轻量信息结构，减少 PC 长句说明。
- 把不重要或低频的功能收进菜单或按钮，不再长期占用主战场。
- 重新规划 5 个出牌区，让桌面构图更规整、更对称。
- 把头像、身份、等级、剩余牌数、个人分数等关键信息并入出牌区头部。
- 简化独立玩家面板，不再承担“展示手牌”的任务。

## 3. 新版布局原则

### 顶部

- 顶部只保留一行高优先级状态：
- 局
- 主
- 朋
- 阶段
- 当前轮到谁
- 倒计时
- 非打家总分

- 顶部状态和右上菜单并入同一个整体容器。
- 倒计时和非打家总分放在顶部最左侧，作为第一视觉信息。
- 计时与总分都用短标签显示，避免顶部出现大段字面说明。
- 朋友牌缩成顶部里的紧凑状态块。
- `游戏状态 / 当前首家` 这类提示移入 `局` 状态块，不再单独挂在游戏名下方。
- `主` 状态块增加主牌可视徽章，和 `朋` 的可视锚点保持统一。
- 顶部去掉游戏标题，只保留状态信息和右上菜单。
- 规则帮助、Debug、播报、底牌、布局编辑、AI 难度、牌面切换等放进 `更多功能` 菜单。
- 顶部不承担新手导引职责，只保留短状态，不放规则解释句。
- 顶部 5 个功能按钮固定为单排图标：`托管 / 信息 / 回看上轮 / 新游戏 / 设置`。
- 顶部 5 个按钮统一使用提供的图标素材，其中 `新游戏` 改用外部提供的 [new_game.png](/Users/mo.li/Documents/cards/icons/icon_new_game.png)。

### 中央战场

- 中央区域作为 PC 第一视觉焦点。
- 5 个玩家出牌区统一为同规格卡片，避免旧版左大右小、上下松紧不一致的问题。
- 每个出牌区头部直接显示：
- 头像
- 玩家名
- 阵营身份
- 等级
- 剩余牌数或个人分数
- 特殊状态，例如 `大`、`庄`、`友`

- 未出牌时显示短空态，例如 `待出`，不再写长说明。
- 其他玩家出牌区标题去掉“出牌区”三个字，只保留玩家名本身。
- 出牌区默认不再显示“本轮尚未出牌”这类空态提示，让中央留白更干净。
- 出牌区里的 `庄 / 朋` 身份改成和手游一致的底色短签，不再用长副标题解释。
- 出牌区里的主牌卡不再使用黄色描边强调。
- 中央保留一条焦点提示带，用于关键操作提示和重要播报。
- 中央桌面背景整体右移，避免和左侧玩家面板重叠。

### 玩家面板

- 玩家面板退化成轻量状态列。
- 5 个玩家面板统一移动到左侧，形成连续纵向状态列。
- 默认只保留头像、序号、身份或轮转信息。
- 去掉“谁操控”这类说明文字。
- 阵营统一改成 `庄 / 闲 / 友` 三种紧凑图标，未知阵营直接留空。
- 左侧玩家面板整体背景改成更轻的淡色玻璃感，不再使用厚重底色。
- 玩家1 面板做成更大的专属卡片，额外显示自己的短状态，而不是沿用其他玩家的同款模板。
- 玩家1 左侧面板最终收成和其他玩家同宽、但高度做成正方形的专属卡片，和玩家2之间的垂直节奏保持一致。
- 不再在独立玩家面板里展示手牌内容。
- 需要展示的关键状态，优先合并到对应的出牌区头部。

### 底部

- 底部改成“左手牌区、右操作区”的双栏结构。
- 左侧整块交给手牌分组展示，取消手牌区里的操作说明文案。
- 底部手牌区去掉头像、标题和 `Lv / 手牌数` 等头部信息。
- 右侧操作区只保留按钮，不放阶段说明和选牌提示文案。
- 各花色分组增加数量徽标，不显示滚动条，改为通过更大的牌面重叠来容纳长手牌。
- 常态下按约 `20% ~ 25%` 的可视牌面密度压叠；拿起底牌后再动态加大重叠，保证 31 张牌仍尽量留在同一视野内。
- 底部手牌区移除整块浅色背景，让牌组直接贴近桌面展示。
- 高频动作继续贴近手牌区右侧保留，形成独立操作区。
- 右侧操作区恢复偏蓝的配色，和中央桌面冷色环境更统一。

### 开始界面

- 开始界面保留 `开始游戏 / 继续游戏 / 查看规则` 三个主入口。
- `AI难度` 从局内顶部移到开始界面设置。
- 开始界面增加主视觉图片，增强入口氛围，风格向手游首页靠拢。

## 4. 与 mobile 的对齐点

- 都把公共状态压缩成短标签和短句，不依赖大段说明文案。
- 都强调中央战场的当前局势，而不是边栏堆信息。
- 都优先展示行动相关信息，再展示解释性信息。
- 都把低频功能收纳，不抢主流程注意力。

## 5. 当前实现状态

- 已落到 PC 运行态页面的内容：
- 顶部整合成一个统一状态条，包含 `局 / 主 / 朋 / 阶段 / 倒计时 / 总分 / 右上菜单`
- ready 阶段独立开始界面
- 开始界面补入主视觉图片和 `AI难度` 选择
- 统一视觉语言的工具图标按钮
- 顶部 `设置` 按钮展开的更多功能菜单
- 玩家面板轻量化
- 左侧玩家面板身份统一改成 `庄 / 闲 / 友` 三种颜色语义，玩家1 面板加大并显示自己的短状态
- 左侧玩家1 面板收成与其他面板同宽的正方形卡片，和玩家2 之间保持统一间隔
- 出牌区头部并入等级、剩余牌数、个人分数和身份信息
- 出牌区头部进一步压缩为必要身份和状态角标，未知阵营直接留空，不再显示 `Lv`
- 其他玩家出牌区标题去掉“出牌区”，默认不再显示“本轮尚未出牌”，并把 `庄 / 朋` 改成手游式底色短签
- 出牌区主牌去掉黄色边框，玩家1 出牌区上移并适度加宽，和其他玩家区形成更明确区分
- 左右两列出牌区重新对齐：`玩家4 / 玩家3` 向外，`玩家5 / 玩家2` 向内
- `无` 和 `大` 角标恢复独立定位，避免头部精简后漂移
- 顶部图标按钮补足统一描边、圆角和显色滤镜，避免图标发白或看起来像空白按钮
- 我的手牌左侧直接贴到桌面最左侧，版本号移到右侧操作区上方
- 中央战场补回 SVG 设计稿里的桌面范围描边和中心多边形底纹
- 中央桌面背景向右收，避免和左侧玩家面板区域重叠
- 底部手牌区改成左手牌、右操作的双栏结构
- 底部手牌区去掉头像、标题和手牌统计头部，并移除整块浅色背景
- 底部手牌区移除滚动条，改用更高重叠密度承载长手牌和拿到底牌后的 31 张牌
- 右侧操作区恢复蓝色主题
- PC 紧凑化改动已加平台隔离，mobile 继续沿用原有出牌区、玩家面板和手牌摘要结构

- 仍可继续迭代的内容：
- 更多功能菜单的进一步分组与收纳
- 出牌区卡片尺寸和位置的进一步精修
- 与 mobile 更进一步统一的动效和切换反馈

## 6. 本轮不包含的内容

- 不修改出牌、叫朋友、扣底、计分等玩法逻辑。
- 不同步改 mobile 实现。
